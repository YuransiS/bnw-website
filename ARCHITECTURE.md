# ARCHITECTURE.md (Жива карта проекту & CRM Blueprints)

## 1. Текущий статус проекта и стек
* **Стек:** Next.js (App Router), Supabase (Database, Auth, RLS), Vercel (Deployment).
* **UI-компоненты:** Tailwind CSS, Lucide Icons. Чистый, премиальный монохромный SaaS-минимализм (Slate/Zinc) с полной поддержкой светлой/темной темы (выбор сохраняется в `localStorage`).
* **Система ролей B&W CRM:**
  - **Супермен (`admin` / `superman`):** Полный доступ ко всем проектам, сводная аналитика всего холдинга, ROI рекламных кампаний, управление сотрудниками, ролями и распределение доступов. **Исключение:** Особый изолированный статус проекта `bw_main` — доступ к нему закрыт даже для Суперменов, если они не привязаны к нему явно в junction-таблице `profile_projects`.
  - **Операционный продюсер (`producer`):** Полный доступ (аналитика, графики, воронка, лиды, канбан) исключительно к закрепленным за ним проектам.
  - **Отдел продаж (`sales`):** Работа с лидами (список, канбан-доска) и генератор платежных кнопок только для разрешенных проектов (высокоуровневая финансовая аналитика и расходы скрыты).
  - **Очікує схвалення (`pending`):** Начальный статус после регистрации. Полный заслон безопасности — отображается только страница ожидания одобрения администратором.

## 2. Файловая структура и компоненты
* `src/app/admin/login/page.tsx` — Экран входа в CRM с поддержкой переключения Вход / Регистрация и Google OAuth авторизацией.
* `src/app/admin/pending/page.tsx` — Защищенный экран ожидания одобрения для новозарегистрированных пользователей с ролью `pending`.
* `src/app/admin/(dashboard)/page.tsx` — Центральный роутер CRM. Принимает параметр `?slug=slug_name` для переключения проекта и рендерит `LeadsDashboard`.
* `src/app/admin/(dashboard)/layout.tsx` — Защищенный макет панели управления, проверяющий авторизацию и перенаправляющий `pending` пользователей на страницу одобрения. Монтирует интерактивный динамический сайдбар `<Sidebar />`.
* `src/app/admin/(dashboard)/Sidebar.tsx` — Интерактивный клиентский компонент сайдбара с возможностью плавного сворачивания (до `72px`), кэшированием состояния в `localStorage`, hover-подсказками (tooltips), разнообразием премиальных иконок для проектов, ссылкой перехода на основной сайт и перенесенными в нижнюю часть настройками профиля.
* `src/app/admin/(dashboard)/settings/page.tsx` — Страница настроек доступов для Суперменов (загружает пользователей, проекты и доступы).
* `src/app/admin/(dashboard)/settings/SettingsForm.tsx` — Форма управления командой: редактирование ролей и чекбоксы проектных доступов (чистый UI без лишних описаний прав).
* `src/app/admin/LeadsDashboard.tsx` — Главный интерфейс CRM. Поддерживает динамические темы (Light/Dark), горизонтальный прокручиваемый таб-бар переключения проектов и вкладки:
  1. **Центр управління** (только Супермен) — сводная таблица всех экспертов холдинга (выручка, расходы, ROI) и кампаниям.
  2. **🏆 Лідери ОП** (только Супермен) — аналитическая таблица операционных продюсеров с расчетом виторга, лидов, CPL, чистой прибыли, ROI по закрепленным проектам и автоматическим определением "ОП месяца" (лидера с короной).
  3. **Сквозна аналітика** (Супермен и Продюсер) — интерактивные SVG Spline Area тренды, CSS-воронка конверсии и UTM-анализ.
  4. **Канбан дошка** — интерактивное управление этапами сделок с помощью HTML5 Drag and Drop API.
  5. **База лідів** — таблица контактов с дедупликацией Disjoint Set Union (DSU), отображением полной истории касаний в виде визуальной хронологической дорожки (таймлайна) при клике на любого клиента в списке, новым фильтром по количеству торканий (touchCountFilter) и постраничной клиентской пагинацией по 100 лидов для максимальной быстродействия.
  6. **Платіжні кнопки** — генератор безопасных WayForPay платежных ссылок.
* `src/app/admin/actions.ts` — Server Actions для выборки объединенной аналитики, создания лидов вручную, обновления статусов в канбане.
* `src/app/admin/(dashboard)/settings/actions.ts` — Server Actions для создания, удаления сотрудников и управления связями в `profile_projects`.
* `src/app/auth/callback/route.ts` — API роут для обмена временного OAuth-кода на сессию Supabase.

## 3. Схема данных (Supabase PostgreSQL)

### Таблица `public.profiles`
* `id` (UUID, primary key references `auth.users`)
* `email` (TEXT)
* `role` (TEXT, по умолчанию `'pending'`, варианты: `'superman'`, `'producer'`, `'sales'`)

### Таблица `public.profile_projects`
Таблица связей (junction table) для распределения доступов сотрудников к проектам.
* `profile_id` (UUID references `public.profiles(id) ON DELETE CASCADE`)
* `project_id` (UUID references `public.projects(id) ON DELETE CASCADE`)
* `PRIMARY KEY (profile_id, project_id)`

### Автоматический триггер `on_auth_user_created`
При любой регистрации пользователя в Supabase Auth автоматически создается профиль со статусом `pending`:
```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, email, role)
  VALUES (new.id, new.email, 'pending')
  ON CONFLICT (id) DO UPDATE SET email = EXCLUDED.email;
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

### Сводные RPC функции
* `public.get_projects_summary()` — Возвращает таблицу агрегированных метрик по всем проектам холдинга (выручка USD/UAH, затраты, уникальные лиды/клиенты и CPL) без ограничения на 1000 строк.
* `public.get_campaigns_summary()` — Возвращает таблицу ROI и окупаемости по рекламным кампаниям холдинга.

## 4. Логика сквозной аналитики и слияния лидов (DSU)
* **Disjoint Set Union (DSU):** Клиентская CRM на лету группирует отдельные заявки, клики и транзакции, разделяющие одинаковый номер телефона, Telegram, Email или `visitor_uuid`, формируя единый профиль клиента с нормализацией статусов (WFP/Google Sheets ➔ 9 CRM stages: от `Новий лід` до `Відмова`), автоматическим промоушеном в `Зацікавлений лід` при 2+ заполнении формы, фильтрацией пустых сессий без контактов, разделением бюджетов на Оплачено и Попытки (Unpaid Intent), отображением всех шагов касания (multi-touch journey) и тегом `"Мульти-канал"`.
* **Сквозная сводка холдинга:** Супермен видит общую окупаемость по экспертам, где расходы подтягиваются из `daily_traffic_and_costs`, а выручка и лиды из `unified_orders` и `unified_customers`.
